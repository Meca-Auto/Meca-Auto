🔒 Guide d'Intégration - Système de Gestion des Cookies RGPD

Ce guide explique comment intégrer le système complet de gestion des cookies conforme RGPD sur votre site Meca Auto.

📋 Étape 1 : Structure des fichiers

Vous avez maintenant 3 nouveaux fichiers dans votre répertoire racine :

  • cookies-consent.css - Styles de la bannière
  • cookies-consent.js - Logique JavaScript
  • COOKIES-INTEGRATION.html - Ce guide (à supprimer après)
✓ Structure créée automatiquement - Aucun dossier à créer

🔧 Étape 2 : Intégration dans index.html

Localisation dans votre fichier :

Dans la balise <head> de votre index.html, ajoutez ces deux lignes juste avant la fermeture </head> :

<!-- Système de gestion des cookies RGPD --> <link rel="stylesheet" href="cookies-consent.css">

Ensuite, juste avant la fermeture </body>, ajoutez cette ligne :

<!-- Initialisation du système de cookies --> <script defer src="cookies-consent.js"></script>
⚠️ Important :
  • Le CSS doit être dans le <head>
  • Le JS doit être avant la fermeture </body>
  • Sans ces deux imports, le système ne fonctionnera pas

📍 Étape 3 : Ajouter le lien "Gérer mes cookies"

Dans votre footer (balise <footer>), le système ajoute automatiquement un bouton "Gérer mes cookies". Sinon, vous pouvez l'ajouter manuellement :

<button class="cookies-manage-link" onclick="window.cookiesConsent.showModal()"> 🍪 Gérer mes cookies </button>

🎨 Étape 4 : Personnalisation (Optionnel)

Modifier les couleurs

Dans cookies-consent.css, les variables CSS en début du fichier contrôlent les couleurs :

:root { --primary-color: #ef4444; /* Rouge - Couleur principale */ --primary-dark: #dc2626; /* Rouge foncé */ --primary-light: #fca5a5; /* Rouge clair */ --text-dark: #1e293b; /* Texte sombre */ --text-light: #64748b; /* Texte clair */ /* ... autres variables ... */ }

Modifier les textes

Editez directement le HTML généré dans la fonction createBanner() et createModal() du fichier cookies-consent.js.

Modifier la langue

Les textes sont en français par défaut. Pour les changer, modifiez les chaînes dans les fonctions createBanner() et createModal().

🔌 Étape 5 : Intégration avec votre code JavaScript

Le système expose l'objet window.cookiesConsent globalement. Vous pouvez donc utiliser ces méthodes :

// Vérifier si l'utilisateur a accepté les analytics if (window.cookiesConsent.isAccepted('analytics')) { // Charger Google Analytics, Matomo, etc. loadGoogleAnalytics(); } // Vérifier le marketing if (window.cookiesConsent.isAccepted('marketing')) { // Charger Facebook Pixel, Google Ads, etc. loadFacebookPixel(); } // Écouter les changements de consentement window.addEventListener('cookiesConsentChanged', (event) => { console.log('Action:', event.detail.action); console.log('Préférences:', event.detail.preferences); // Recharger les scripts si nécessaire if (event.detail.preferences.analytics) { loadGoogleAnalytics(); } });

Exemple complet : Google Analytics

// Chargement conditionnel de Google Analytics if (window.cookiesConsent && window.cookiesConsent.isAccepted('analytics')) { // Google Analytics window.dataLayer = window.dataLayer || []; function gtag(){dataLayer.push(arguments);} gtag('js', new Date()); gtag('config', 'GA_MEASUREMENT_ID'); // Chargement du script GA const script = document.createElement('script'); script.async = true; script.src = 'https://www.googletagmanager.com/gtag/js?id=GA_MEASUREMENT_ID'; document.head.appendChild(script); }

🧪 Étape 6 : Test du système

Pour tester rapidement :

  1. Ouvrez votre site dans le navigateur
  2. Une bannière colorée devrait apparaître en bas de page
  3. Cliquez sur "Personnaliser" pour voir la modale
  4. Testez les toggles et les boutons
  5. Inspectez localStorage dans DevTools (F12 → Application → Local Storage)

Réinitialiser pour retester :

Dans la console (F12), tapez :

window.cookiesConsent.reset()

Cela supprimera les préférences sauvegardées et recharger la page.

⚖️ Étape 7 : Conformité RGPD

✓ Points de conformité respectés :
  • ✓ Consentement explicite - L'utilisateur doit cliquer pour accepter
  • ✓ Opt-out facile - Bouton "Refuser tous" aussi visible que "Accepter"
  • ✓ Granularité - Possibilité de choisir catégorie par catégorie
  • ✓ Transparence - Descriptions claires de chaque type de cookie
  • ✓ Stockage - Préférences sauvegardées dans localStorage (pas de cookie)
  • ✓ Durée - Consentement valide 12 mois (modifiable)
  • ✓ Accessibilité - Lien vers politique de confidentialité
  • ✓ Réversibilité - L'utilisateur peut changer ses préférences à tout moment

📄 Mettez à jour votre politique de confidentialité

Assurez-vous que votre politique-confidentialite.html contient une section "Cookies" expliquant :

  • Quels cookies vous utilisez
  • À quoi ils servent
  • Comment les utilisateurs peuvent les contrôler

🐛 Troubleshooting

La bannière n'apparaît pas

  • Vérifiez que le CSS est chargé (F12 → Network)
  • Vérifiez que le JS est chargé (F12 → Network)
  • Vérifiez la console (F12 → Console) pour les erreurs
  • Réinitia avec window.cookiesConsent.reset()

Le localStorage ne sauvegarde pas

  • Vérifiez que le navigateur n'est pas en mode privé
  • Vérifiez que localStorage est activé (F12 → Application)
  • Vérifiez la clé stockée : meca-auto-cookies-consent

Les styles ne s'appliquent pas

  • Vérifiez que le CSS est chargé avant votre CSS personnalisé
  • Vérifiez qu'il n'y a pas de conflit avec Tailwind CSS
  • Utilisez l'inspecteur CSS (F12) pour voir les styles appliqués

Le formulaire de contact charge les cookies

  • Si votre formulaire utilise un service externe (Formspree, Netlify, etc.), assurez-vous de charger le script après le consentement
  • Vérifiez que le action du formulaire n'appelle pas d'API non consentie
